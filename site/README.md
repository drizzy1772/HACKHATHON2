# AI SkyRun Hackathon — інтерактивний сайт змагання

Статичний self-contained сайт гейміфікації: дерева ачівок (Minecraft-style),
система SkyPoints і лідерборд. Без збірки, без залежностей — чистий HTML/CSS/JS.

## Запуск

Через локальний сервер (браузери блокують `fetch` з `file://`):

```bash
cd site
python3 -m http.server 8000
# відкрий http://localhost:8000
```

Жодного `node`/`npm`/build не потрібно. Працює офлайн.

## Структура

```
site/
  index.html            лендинг: 4 плитки-треки (вибір напрямку) + лідерборд
  tier.html             сторінка тіра: дерево ачівок (?tier=D|C|B|A)
  css/skyrun.css        палітра з tier_d/viz/style.py; light/dark; кольорові статуси
  js/theme.js           перемикач теми (light/dark, локально)
  js/scoring.js         формули SkyPoints (авто-бали + ачівки + престиж-множник)
  js/tree.js            рендер дерева, розблокування (parents), панель, фільтр, прогрес
  js/leaderboard.js     плитки-треки + глобальний/потрековий лідерборд
  data/achievements.json  УВЕСЬ контент: 4 тіри, ~48 ачівок з абзацами, кольори, бали
  data/teams.json         зразковий ростер команд (редагується під час події)
```

## Формат змагання

**Вибір треку.** Кожна команда обирає ОДИН напрямок і йде вглиб його дерева:

| Трек | Рівень | Множник | Задача |
|---|---|---|---|
| **D** | EASY | ×1.0 | Табличний Q-learning на сітці |
| **C** | MEDIUM | ×1.4 | Класична навігація A*+APF (Blender) |
| **B** | HARD | ×1.8 | CTBR PPO vs SAC, справжня фізика |
| **A** | UNKNOWN | ×2.2 | 6DOF attitude, дослідницький «чесний 0/10» |

## Скоринг (гібрид)

Внутрішньотрековий бал = **SP_auto** (авто-метрики) + **SP_achievements** (ачівки).
Глобальний бал = `(SP_auto + SP_achievements) × престиж-множник` — складніший трек престижніший.

- **SP_auto** — з формул `js/scoring.js`, дзеркалять scoring-harness кожного тіра
  (D: success/collision/L*L/eff · C: статус+час+гладкість · B: рівень+крен · A: RMSE+bounty).
- **SP_achievements** — сума SP розблокованих ачівок.

## Кольорові статуси задач

| 🟢 green | 🟡 yellow | 🟣 purple | 🔴 red | 🔵 blue |
|---|---|---|---|---|
| відтворити | вдосконалити | концептуально змінити | відкриває нові проблеми | ідеал / compute-ризик Mac-mini |

Вищий ризик = вища винагорода (`red`, `blue` дають найбільше SP).

## Суддівський флоу

1. Команда працює у своєму треку в основному репозиторії (див. `prompts/tier_*.md`).
2. Авто-бали: організатор запускає scoring-harness тіра, вписує метрики в
   `data/teams.json` → поле `auto` (формат полів — під формулу треку в `scoring.js`).
3. Суддівські бали: за виконану ачівку суддя додає її `id` у `unlocked` команди.
4. Лідерборд перераховується сам при перезавантаженні сторінки.

Приклад запису команди в `data/teams.json`:

```json
{ "id": "t-d1", "name": "Blind Navigators", "track": "D",
  "auto": { "success": 1.0, "collision": 0.0, "optimality": 0.96, "sample_eff": 0.82 },
  "unlocked": ["d-first-flight", "d-bellman", "d-shaping", "d-tuned"] }
```

## Прогрес учасника (демо розблокування)

На сторінці тіра клік по ноді відкриває пояснення. Кнопка «Позначити виконаним»
демонструє логіку розблокування (parent→child) і рахунок SP — зберігається
локально в браузері (`localStorage`), окремо від офіційного `teams.json`.

## Цілісність контенту / admin-safety

`data/achievements.json` описує МЕТУ кожної ачівки, а не розв'язок: без
hidden-сідів, reference-коду й admin-паролів (поважає інваріант 5 «hidden ≠ public»
та секретність `*/admin/` з prompt-карток). Перевірка цілісності (валідні кольори,
parents існують, немає циклів) — у процесі складання контенту.
