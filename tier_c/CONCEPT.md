# Tier-C · «Політ крізь ліс» — реалістичний FPV-дрон у Blender

## Концепція

Автономна навігація квадрокоптера крізь процедурний ліс у повному 3D.
Арена 50×50 м із рельєфом (Perlin-шум), деревами, тематичними перешкодами і
вантажівкою-чекпоінтом у дальньому куті. Дрон стартує в `md.start` і має
долетіти до `md.checkpoints[0]`, оминаючи дерева, **не піднімаючись** вище
стелі (середня висота крон — вище = «Під дією РЕБ», дискваліфікація) і **не
торкаючись** рельєфу (контурний політ). Ліміт 90 с, цикл керування 30 Гц.

Статуси прогону: `FINISHED` / `COLLISION` («Борт втрачено») /
`DISQUALIFIED` («Під дією РЕБ») / `TIMEOUT`.

На відміну від tier_d (табличний RL на сітці) це трек **класичної
навігації**: глобальний планувальник (A*) + реактивне ухилення (APF за
даними лідара) + кінематичний автопілот. RL тут немає — є геометрія,
вектори і фізична інтуїція.

## Контракт учасника

Команда редагує **ЛИШЕ** `tier_c/solution.py`. Вузький контракт:
**(lidar_data, current_position, next_checkpoint) → (vx, vy, vz)**,
розгорнутий у чотири функції (усі зараз — безпечні заглушки: дрон висить):

| що | сигнатура | коли викликається |
|---|---|---|
| глобальний маршрут | `find_path(md, cfg, cell_size=1.0) -> Optional[List[Vec2]]` | один раз на старті |
| реактивний напрямок | `compute_desired_direction(pos, lidar, bin_angles, tracker, stuck, t, p, boost_state) -> (dir_xy, boosted, carrot)` | щотік, 30 Гц |
| детекція застрягання | `StuckDetector.update(t, x, y) -> bool` | щотік |
| виконавець | `step_autopilot(state, direction_xy, terrain, dt, p, min_lidar=None) -> AutopilotState` | щотік |

Плюс налаштовні дата-класи `APFParams` і `AutopilotParams`.
Дано (не змінювати): `astar2d.py` (occupancy grid, конвертації), `apf_controller.py`
(`PathTracker` — carrot-chasing), `kinematic_autopilot.py` (контракт `AutopilotState`),
`game_env/` (генератор мап із сідом, лідар, рельєф, сцена).

## Запуск

```bash
# headless-самоперевірка (numpy, без Blender): рахує траєкторію → out/autonomous_runs.json
python tier_c/sim_headless.py

# повна гра (ручний політ + автономний режим, 3D):
/Applications/Blender.app/Contents/MacOS/Blender -P tier_c/blender_manual.py
# або подвійний клік:  tier_c/Запустити гру.app

# HTML-звіт метрик останнього автономного прогону
python tier_c/metrics_report.py
```

## Адмін/учень

- **Учасник**: редагує лише `solution.py`; решта — read-only.
- **Організатор**: `tier_c/admin/solution.py` — робочий референс усіх чотирьох
  функцій + еталонна APF-математика для адмінської теплокарти; там само живе
  пароль адмін-режиму Blender-гри. `tier_c/admin/` **не роздається**: без цієї
  теки гра працює як учасницька (без адмін-режиму і теплокарти), а
  `blender_manual.py` деградує граційно.

## Оцінювання

Прогін на сідах журі (сід мапи фіксує ліс/рельєф/чекпоінт byte-у-byte):
статус фінішу, час до чекпоінта, кількість застрягань, гладкість траєкторії —
`metrics_report.py` генерує звіт по останньому прогону.
