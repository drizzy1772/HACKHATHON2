# Tier-B — «CTBR Pirouette»: концепт завдання

## Що це за трек

Прямий наступник Tier-A: там було виміряно, що з фіксованою hover-тягою
пірует фізично неможливий. Tier-B знімає це обмеження — дія агента стає
**CTBR**: `[колективна тяга c ∈ [0, 2g], швидкості обертання ω_x, ω_y, ω_z]`.
Стек: справжня фізика **gym-pybullet-drones (PyBullet)**, дрон Crazyflie 2.X,
два алгоритми **PPO і SAC** (stable-baselines3) на спільному середовищі з
однаковими сідами лісів — чесне порівняння за конструкцією.

Задача: пролетіти 12 м крізь три «ворота» (пари дерев) із щілиною, що
звужується курикулумом від 1.20 м до 0.12 м — при ширині корпусу 0.24 м.
Верхні рівні прохідні ЛИШЕ з креном (еліпсоїд 12×12×4 см тоншає у профілі
як w(φ) = √((a·cosφ)² + (c·sinφ)²)).

## Що вже зроблено (не переробляти — відтворювати)

- CTBR-конвеєр (rate-P контролер + міксер CF2X), валідований відкритим
  контуром ДО навчання; 34-променевий LIDAR (8+6+4+16, точна конструкція в
  `envs/lidar.py`); аналітична еліпсоїд-колізія з глибиною проникнення;
  reward із potential shaping; курикулум на 9 рівнів; повний тренувальний
  стек, eval на прихованих сідах, порівняння, відео, Blender-експорт.
- **Головний результат**: SAC за 2M кроків сам відкрив прохід щілини 0.26 м
  із креном **35–49°** (виміряно в успішних прольотах) — маневр, недосяжний
  у Tier-A. PPO дійшов до L3 (0.40 м, рівний політ).
- **Три виміряні reward-хаки** та їхні ліки — задокументовані у README і
  закріплені тестами (обов'язково прочитати перед зміною винагороди!).
- Зонд `probe_flick.py`: три виміряні закони динаміки крутого крену
  (момент потребує тяги; бічний імпульс Δv_y = c(1−cosφ)/ω̄ залежить лише
  від кінцевих кутів; висота — дешевий ресурс).

## Що тут потрібно зробити (відкрита частина завдання)

Рівні **L6–L8 (0.20 / 0.16 / 0.12 м, крен 45–75°) не скорені** — це чесна
стеля бюджету 2M кроків, а не межа методу. Конкретні напрямки:

1. **Продовжити навчання SAC** із warm-start від чекпоінта L5
   (`runs/sac/seed0/ckpt/`) на 5–10M кроків.
2. **Статистика**: кілька незалежних сідів (скрипти вже multi-seed-ready:
   `--seed`), звіт per-seed у стилі репозиторію (0/N — теж результат).
3. **Reward-інженерія**: м'яке заохочення запасу до перешкоди (обережно —
  спершу README-розділ про хаки), або окремий бонус за крен у зоні воріт
  (ризик: пряма підказка ламає чистоту експерименту «відкрив сам»).
4. **Навичка з зонда**: краб-підхід → S-мах → балістичний прохід →
   відновлення. RL мусить відкрити її сам; можна допомогти курикулумом
   по швидкості підходу.

## Як запускати

```bash
conda run -n drones python -m pytest tier_b/admin/tests        # 67 тестів
conda run -n drones python tier_b/train_ppo.py           # 2M, ~1 год (M4)
conda run -n drones python tier_b/train_sac.py           # 2M, кілька годин
conda run -n drones python tier_b/compare.py             # PPO vs SAC → tier_b/admin/reports/
conda run -n drones python tier_b/eval.py --model tier_b/runs/sac/seed0/final_model.zip --level 5
conda run -n drones python tier_b/viz_lidar.py --model ... --level 5   # живий GUI
```

Установка середовища і всі деталі — `tier_b/README.md`.

## Непорушні правила

- torch / gym-pybullet-drones / SB3 живуть ЛИШЕ в conda-env `drones`;
  кореневий `pytest` основного кіта має лишатися зеленим без них
  (importorskip-гварди вже стоять).
- Оцінювання — тільки на held-out сідах (offset 900000+), ніколи не бачених
  у тренуванні.
- Обидва алгоритми завжди порівнюються на однакових сідах лісів.
- Чесні звіти: негативний або частковий результат публікується з діагнозом,
  а не ховається.

## Учнівська частина (scaffold) і скоринг

Роздача командам = `tier_b/` **без** `tier_b/admin/`. Команда заповнює дві діри:

1. `scaffold/student_rewards.py` — `shaping()` та `step_reward()` (інваріант 4:
   `F = γ·Φ(s′) − Φ(s)`, `Φ(термінал) = 0`);
2. `scaffold/student_config.yaml` — гіперпараметри PPO/SAC (усі `null # TODO`).

`tier_b/rewards.py` — диспетчер: в організаторській копії бере референс
`admin/rewards_solution.py`, в учнівській (без admin/) — автоматично
`scaffold/student_rewards.py`; примусово: `TIER_B_STUDENT_REWARDS=1`.

Самоперевірка команди: `conda run -n drones python tier_b/scaffold/validate_team_b.py
--model <ваш final_model.zip>` (PUBLIC_SEEDS = held-out 900000+). Фінальний залік:
`python -m tier_b.admin.scoring.harness_b --model ...` на HIDDEN_SEEDS
(`admin/scoring/hidden_seeds.py`, не роздається). Еталон розв'язку регенерується
`python tier_b/admin/make_solution.py` і звіряється тестом
`admin/tests/test_scaffold_b.py` — розв'язок ніколи не дрейфує від тестів.
