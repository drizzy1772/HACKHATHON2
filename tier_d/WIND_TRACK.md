# Wind Track — Controlled Comparison

Параллельний трек для вивчення впливу стохастичності на RL.

## Структура

```
tier_d/env/                          (базовий)
├── constants.py              (WIND_PROB = 0.0)
├── gridworld.py              (детерміністичний)
└── ...

tier_d/env_wind/                     (копія з вітром)
├── constants.py              (WIND_PROB = 0.05)
├── gridworld.py              (GridWorldWind з вітром)
└── ... (решта ідентична)

tier_d/scaffold/starter.ipynb        (базовий ноутбук)
tier_d/scaffold/starter_wind.ipynb   (wind-версія ноутбука)

tier_d/experiments/
├── compare_wind.py           (side-by-side тренування)
└── README.md
```

## Як користуватися

### Для командопереглядку

**Базовий track (як завжди):**
```bash
jupyter notebook tier_d/scaffold/starter.ipynb
```

**Wind track (для вищого рівня):**
```bash
jupyter notebook tier_d/scaffold/starter_wind.ipynb
```

Ноутбуки ідентичні, крім вирізу вітру в середовищі.

### Для порівняння

```bash
python tier_d/experiments/compare_wind.py --seed 0 --episodes 3000
```

**Вихід:** 
- `tier_d/reports/wind_comparison.png` — 4 графіки
  1. Крива базова (детерміністична)
  2. Крива wind (стохастична)
  3. Обидві разом
  4. Таблиця: успіх, L, L*/L, різниця

## Що показує експеримент

| Метрика | Базова | Wind | Висновок |
|---------|--------|------|----------|
| Швидкість учення | ✓✓✓ швидка | ✓ повільна | Вітер замовляє учення |
| Гладкість кривої | гладка | шумна | Стохастичність = шум |
| Фінальний L | низький | вищий | Вітер коштує довідпорів |
| Вміст | запам'ята | адаптується | Wind = робастність |

## Для учнів 11-го класу

**Чи варто спробувати wind?**

✅ Якщо:
- Базовий трек пройшов легко (success > 90%)
- Вас цікавить реальна робастика (дрони мають помилки)
- Вибір тюнінгу для роботи на більш складних умовах

❌ Якщо:
- Базовий трек ще не пройшов
- Вам потрібна просто хорошая оцінка

## Для організаторів

**Використання на змаганні:**

День 1: базовий track (для всіх)
День 2: wind track (для лідерів лідієрбарду)
День 3: порівняння обох, журі оцінює методику вибору

**Очікувані результати:**
- 80% команд успішні на базовому середовищі
- 20% спробують wind і матимуть цікаву презентацію про "вартість робастності"

## Деталі реалізації

### WIND_PROB = 0.05

Кожного кроку агент: 
- 95% часу: робить задану дію
- 5% часу: випадкова дія

Це **дуже помітно**, але не сокрушає завдання. Агент все ще може навчитись.

### A* оракул бачить вітер?

Поточна реалізація: **НІ** (L* обчислюється для детерміністичного шляху).

Якщо хочете справедливості, можна переробити оракул на стохастичний A* із ймовірностями переходів. Це складніше і запропоновано як Tier 3 для самостійного вивчення.

## Що експеримент вчить RL

1. **Реальність складніша за теорію.** Без стохастичності RL — це пошук по таблиці. З вітром — справжня адаптація.
2. **Робастність коштує.** Детерміністичний агент заучує оптимальний шлях. Стохастичний вчиться адаптуватися, але це повільніше й дає довший шлях.
3. **Мотивація для глибшого RL.** Таблиця Q плиху справляється зі стохастичністю. Функція наближення (нейромережа) справляється краще.
