## Рівень 0 — базові TODO

Ви маєте **три дірки** в ноутбуці:

### TODO 1: Правило Беллмана
```python
def q_update(Q, s, a, r, s2, terminal, alpha, gamma):
    if terminal:
        target = r                                        # у терміналі нема майбутнього
    else:
        target = r + gamma * np.max(Q[s2])               # інакше дисконтуємо max Q
    td = target - Q[s, a]
    Q[s, a] = Q[s, a] + alpha * td
    return td
```

**Що буває, якщо заплутали:**
- `target = r + gamma * Q[s2, a]` (переносите дію) → агент вчиться повільно
- `target = r + gamma * max(Q[s2])` навіть у терміналі → стрибки на закінченні, гарантія shaping зникає

## Рівень 1 — стандартний RL

Базові TODO готові, крива піднялась на базовому середовищі. Тепер — тюнінг.

### Питання: як визначити `ALPHA` й `GAMMA`?

**ALPHA** контролює, наскільки агент довіряє новому досвіду:
- `0.05`: дуже консервативно, збіжність повільна, але стабільна (особливо для нестійких бенчмарків)
- `0.15`: баланс (рекомендується)
- `0.3`: агресивна, швидше, але шуму більше

**Вибір:** спробуйте спочатку `0.15`. Якщо крива стрибає, зменште на 2–3×. Якщо «важко» (багато епізодів), збільште на 1.5×.

**GAMMA** дизайнує горизонт:
- `0.9`: окремі кроки важлива, глобальна стратегія менше. Горизонт ≈ 10 кроків.
- `0.99`: видалена ціль важлива. Горизонт ≈ 100 кроків. Рекомендується для сітки 25×25.
- `0.999`: дуже далекоглядна, але збіжність гірша.

**Вибір:** для вас `0.99` майже завжди правильний.

### Де вилізає `L*`?

```python
from oracle.astar import optimal_length
L_star = optimal_length(env.trees)
```

Це A*-оракул бачить дерева й рахує найкоротший шлях. Ви цього шляху не знаєте,
але **не можете його перебити** — дрон летить по сітці, а оракул — по колінійним сегментам.

Тому `efficiency = L*/L` **ніколи не буває > 1**. Якщо ви це бачите — баг у кіті.

У вас лишився час? Спробуйте одне з цього:

### 2a. Інший потенціал `Φ`

У нас: `Φ(s) = −dist(s, B)`. Чому не Манхеттен?

```python
def potential_manhattan(s):
    r, c = divmod(s, GRID_N)
    gr, gc = GOAL_CELL
    return -(abs(r - gr) + abs(c - gc)) * CELL
```

Евклід дає опуклу поверхню (М-м, краще); Манхеттен — хрестову (теж робить, але інакше).
**Тест:** замініть у `shaping_F` й дивіться криву.

### 2b. Оптимістична ініціалізація

```python
Q = np.zeros((env.n_states, N_ACTIONS))
# замість цього:
```

Це дозволяє агенту дослідити невідомі дії більш охоче (кожна видається потенційно хорошою).

**Тест:** спробуйте на тому ж seed, порівняйте кривих.

## Як читати табло (scoring.harness)

Після фіналу вас оцінюють на **прихованих** seed-ах.

```
seed   L*      L      L*/L  success  collisions
  0   11.30   12.03   0.939   ✓        0
  1   12.60   13.10   0.962   ✓        0
  2   13.05   15.12   0.863   ✓        1
  ...
avg      —     12.95   0.927   100%     0.1
```

**Ранг:** лексикографічний (по порядку):
1. **Success** — % команд, які дійшли. Не дійшли — 0 балів, кінець.
2. **Collision rate** — дійшли, але б'ються? → теж погано.
3. **Optimality L*/L** — наскільки близько до A*. **0.85–1.00 це нормально.**
4. **Sample efficiency** — за скільки епізодів досяг `1.15·L*`.

**Чому не просто L?** Тому що L залежить від seed-у. Дерево ставитьсяпо-різному —
іноді ліс «щільний» (потрібно більше кроків), іноді розрідженій. Нормалізація на L*
робить порівняння справедливим.
