# Tier-D — базовий 2D-кіт: концепт завдання

## Що це за трек

Фундамент усього хакатон-кіта і **єдиний змагальний трек для команд**.
Світ — плоска сітка 25×25 над доменом 10×10 м із seeded-лісом
(`mulberry32`: однакові `(n, seed)` → байт-у-байт однаковий ліс у Python і
в JS-візуалізаторі `viz/field_scanner.html`). Завдання команди — написати
**табличний Q-learning** агент, який долітає з A(1,1) до B(9,9), і
оцінюється проти A*-оракула: `efficiency = L*/L ≤ 1`.

Це «вивчення у 2D»: жодної фізики польоту, жодних нейромереж, жодного GPU —
дрон «телепортується» між клітинками, і вся складність — у чистому RL:
розвідка проти використання, shaping, збіжність, sample efficiency.

## Склад

| тека | що всередині |
|---|---|
| `env/` | MDP: ліс, сітка, зайнятість (єдине місце з `TREE_R`), 2D LIDAR |
| `env_wind/` | варіант із 5% стохастичних поривів — абляція «ціна стійкості» |
| `oracle/` | A* (октіль, без зрізання кутів), `L*`, feasibility, value iteration |
| `agent/` | еталонний Q-learning + потенціальний shaping |
| `apf/` | класичні потенціальні поля — **контрольований контрприклад**: читає мапу (навмисне порушення інваріанта 2), стоїть у локальних мінімумах на ~34% лісів; ніколи не оцінюється проти команд |
| `scoring/` | precompute кешу L*, лідерборд на прихованих сідах |
| `viz/` | усі фігури, 3D-льодяник лісу, field scanner (HTML/JS), плівка навчання |
| `scaffold/` | стартовий ноутбук команди + validate_team.py + гайди |
| `webots/` | міст політика→вейпоінти для 3D-демо (симулятор не в репо) |
| `tests/` | 125 тестів, включно з шістьма інваріантами |

## Що тут потрібно зробити (для учасника)

1. Прочитати `scaffold/README.md` і `scaffold/GAMEPLAY_GUIDE.md`.
2. Заповнити TODO у `scaffold/starter.ipynb` (правило оновлення Q, ε-розклад).
3. Самоперевірка: `python tier_d/scaffold/validate_team.py`.
4. Оцінка організатором: `python -m tier_d.scoring.harness` (приховані сіди).
5. Опційно: вітряний трек (`scaffold/starter_wind.ipynb` +
   `python -m tier_d.experiments.compare_wind`) — виміряти ціну стійкості.

## Непорушні правила (шість інваріантів, усі під тестами)

1. Відтворюваність: `mulberry32`, Python ≡ JS.
2. Стан = досвід, не мапа (агент бачить лише індекс клітинки).
3. Один `TREE_R` — тільки в `env/occupancy.py`.
4. Shaping лише потенціальний, `Φ(термінал)=0`.
5. Приховані сіди ніколи не в тренуванні.
6. `V(s)` — navigation function: один мінімум, у цілі.

## Звідки решта тірів

Цей кіт свідомо обходить питання «як фізично летіти» (`webots/bridge.py`
залишає орієнтацію «існуючому PID»). Саме цю дірку закриває **Tier-A**
(6DOF attitude control, фіксована тяга → фізичне вето піруета), а його
вето породило **Tier-B** (CTBR + PPO/SAC, пірует виконано). Історія і
команди — у `CONCEPT.md` відповідних тек.
